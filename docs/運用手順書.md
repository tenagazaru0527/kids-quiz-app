# クイズひろば 運用手順書(おうちの人向け)

家族用クイズアプリ「クイズひろば」のセットアップと日々の運用の手順です。

## 全体のしくみ

```
Google Drive                GAS Web アプリ              GitHub Pages の PWA
┌─────────────┐   読み取り  ┌──────────────┐   fetch   ┌──────────────┐
│ quizzes/     │ ─────────→ │ JSONをまとめて │ ────────→ │ iPad のアプリ  │
│  ├ いきもの/ │            │ 配信 (doGet)  │           │ (ホーム画面)  │
│  ├ うちゅう/ │            └──────────────┘           └──────────────┘
│  └ さんすう/ │
└─────────────┘
```

- **クイズの追加・修正は Google Drive に JSON を置くだけ**。アプリの再デプロイは不要。
- サブフォルダ名がそのままアプリの「ジャンル」になる。フォルダを増やせばジャンルが増える。
- 反映まで最大5分(GAS側キャッシュ)+ アプリを開き直すだけ。

---

## 初回セットアップ

### 1. Google Drive にフォルダを作る

1. [Google Drive](https://drive.google.com) を開き、マイドライブ直下に **`quizzes`** という名前のフォルダを作る(名前は正確に。小文字)。
2. その中にジャンル名のサブフォルダを作る(例:`いきもの`、`うちゅう`、`さんすう`)。
3. このリポジトリの `samples/` にあるサンプル JSON をそれぞれのフォルダにアップロードして動作確認に使う。
   - `doubutsu.json` → `いきもの/`
   - `uchuu.json` → `うちゅう/`
   - `keisan1.json` → `さんすう/`

### 2. GAS(Google Apps Script)をデプロイする

1. [script.google.com](https://script.google.com) を開き、「新しいプロジェクト」を作成。
2. プロジェクト名を「クイズ配信」などに変更。
3. エディタの `コード.gs` の中身を全部消し、このリポジトリの **`gas/Code.gs`** の内容を貼り付けて保存(Ctrl+S / ⌘S)。
4. **動作確認**: 上部の関数選択で `checkQuizzes` を選び「実行」。
   - 初回は権限の承認画面が出る:「権限を確認」→ 自分のアカウントを選択 →「詳細」→「(プロジェクト名)に移動」→「許可」。
   - 実行ログに「クイズ数: 3」「エラーなし ✓」のように出れば OK。
5. **デプロイ**: 右上「デプロイ」→「新しいデプロイ」
   - 種類の選択(歯車アイコン)→「**ウェブアプリ**」
   - 次のユーザーとして実行: **自分**
   - アクセスできるユーザー: **全員**(← 重要。これにしないと iPad から読めない)
   - 「デプロイ」を押し、表示された **ウェブアプリの URL**(`https://script.google.com/macros/s/…/exec`)をコピー。
6. コピーした URL をブラウザで開き、クイズの JSON が表示されることを確認。

> ⚠️ 「アクセスできるユーザー: 全員」にすると、URL を知っている人は誰でもクイズ JSON を見られます。個人情報などは書かないでください。

### 3. アプリに GAS の URL を設定してデプロイする

1. このリポジトリの [src/config.js](../src/config.js) を開き、コピーした URL を貼る:
   ```js
   export const GAS_URL = "https://script.google.com/macros/s/xxxxx/exec";
   ```
2. GitHub にリポジトリを作って push する(初回のみ):
   ```bash
   git init && git add -A && git commit -m "initial"
   git branch -M main
   git remote add origin https://github.com/<ユーザー名>/<リポジトリ名>.git
   git push -u origin main
   ```
3. GitHub のリポジトリページ →「Settings」→「Pages」→ Build and deployment の **Source を「GitHub Actions」** にする。
4. push すると自動でビルド&デプロイされる(`.github/workflows/deploy.yml`)。数分後に
   `https://<ユーザー名>.github.io/<リポジトリ名>/` で開ける。

### 4. iPad のホーム画面に追加する

1. iPad の **Safari** で上記 URL を開く(Chrome ではなく Safari で)。
2. 共有ボタン(□に↑)→「**ホーム画面に追加**」→「追加」。
3. ホーム画面の「クイズひろば」アイコンから起動すると、全画面のアプリとして動く。
4. 一度読み込んだクイズは iPad 内に保存されるので、オフラインでも前回のクイズで遊べる。

## 日々の運用:クイズを追加する

1. パソコンやスマホでクイズの JSON ファイルを作る(下の形式を参照。ChatGPT や Claude に「この形式で○○のクイズを作って」と頼むのが楽)。
2. Google Drive の `quizzes/ジャンル名/` フォルダにアップロードする。
   - 新しいジャンルにしたければ、新しいサブフォルダを作ってそこに入れるだけ。
3. **5分待つ**(GAS のキャッシュが切れる)か、GAS の URL に `?refresh=1` を付けて一度ブラウザで開く。
4. iPad でアプリを開き直す(起動中なら一度閉じて再度開く)。新しいクイズに **NEW!** バッジが7日間付く。

### クイズ JSON の形式

ファイル名は自由(`.json` で終わること)。1ファイル = 1クイズ。

```json
{
  "title": "どうぶつクイズ",
  "emoji": "🦒",
  "difficulty": 2,
  "createdAt": "2026-07-01",
  "questions": [
    {
      "q": "問題文",
      "choices": ["選択肢1", "選択肢2", "選択肢3", "選択肢4"],
      "answer": 0,
      "explanation": "解説文(答えのあとに表示される)"
    }
  ]
}
```

| 項目 | 説明 |
|---|---|
| `title` | クイズの名前(必須) |
| `emoji` | カードに出る絵文字(省略時 ❓) |
| `difficulty` | 1〜3。★の数=むずかしさ。アプリの「しぼりこみ」に使われる(省略時 1) |
| `createdAt` | `"2026-07-01"` 形式。NEW! バッジと並び順に使う(省略時はファイルのアップロード日) |
| `questions` | 問題の配列。1問以上 |
| `q` | 問題文(必須) |
| `choices` | 選択肢 2〜4個(必須) |
| `answer` | 正解の番号。**0はじまり**(0=①、1=②、2=③、3=④)⚠️ 間違えやすいので注意 |
| `explanation` | 解説文 |

- `category`(ジャンル)は**フォルダ名から自動で付く**ので書かなくてよい。
- 1つのファイルに `[ {...}, {...} ]` と配列で複数クイズを入れることもできる。

### 形式ミスのチェック

JSON が壊れていたり `answer` が範囲外だったりすると、そのクイズは**黙ってスキップ**されます(アプリは壊れない)。追加したのにアプリに出ないときは:

1. [script.google.com](https://script.google.com) で GAS プロジェクトを開く。
2. 関数 `checkQuizzes` を実行 → ログにエラーのあるファイルと理由が出る。

## トラブルシューティング

| 症状 | 対処 |
|---|---|
| 「クイズがよみこめなかったよ」と出る | iPad がネットにつながっているか確認 →「もういちど よみこむ」。初回起動は必ずネット接続が必要 |
| 詳細に「GASのURLが設定されていません」 | `src/config.js` の `GAS_URL` を設定して push し直す |
| 追加したクイズが出てこない | ①5分待つ or GAS URL に `?refresh=1` を付けて開く ② アプリを完全に閉じて開き直す ③ `checkQuizzes` で形式ミスを確認 |
| GAS の URL を開くとエラーページ | デプロイ設定の「アクセスできるユーザー」が「全員」になっているか確認。Code.gs を修正したら「デプロイ」→「デプロイを管理」→ 鉛筆 → バージョン「新バージョン」で**再デプロイが必要**(URL は変わらない) |
| アプリの見た目を変えたのに反映されない | push 後、GitHub の Actions タブで緑✓を確認 → iPad でアプリを閉じて2回開き直す(1回目で裏側の更新を取得) |
| ジャンルのフォルダを消したのに残って見える | GAS キャッシュ(最大5分)+ iPad 内キャッシュ。時間をおいて開き直す |

## 運用のコツ

- **クイズ作りは AI に頼む**: `samples/doubutsu.json` を AI チャットに貼って「この形式で『からだのふしぎ』クイズを★2で5問作って。answerは0はじまり」と頼むと速い。できた JSON は一度 `checkQuizzes` で確認。
- **古いクイズの整理**: Drive でファイルを削除・移動するだけ。アーカイブ用に `quizzes` の外へ移せばアプリから消える。
- **難易度の目安**: ★1=低学年でも解ける、★2=小学生なら考えれば解ける、★3=大人も悩む。早押し大会は★を混ぜると盛り上がる。
